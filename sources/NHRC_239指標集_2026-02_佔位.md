# NHRC 兒童權利監測指標集(2026-02-04 發布,239 項)

> **平台對接狀態**:🟡 基礎建設已就緒 / 等待人工匯入指標數值
>
> **NHRC 官方來源**:2026 年 2 月 4 日發布之《兒童權利監測指標集》共 239 項指標,以 SDGs-CRC 為主架構,加入國際審查專家結論性意見及 NHRC 獨立評估意見,涵蓋 SDGs、受教育權、公共事務參與權、兒童司法人權四大支柱。

## 一、指標集結構(對應 MVP 藍圖 §3)

依藍圖第 53 行所述:

> 「建議平台在每個議題卡片下方,直接顯示 NHRC 對應指標的官方數值與更新日期 — 這是 NHRC 自身網站尚未具備的視覺化能力,屬平台的關鍵差異化價值。」

四大支柱(根據 NHRC 公開摘要推估):
1. **SDGs 兒童相關指標**(約 60-80 項)
2. **受教育權指標**(約 50-70 項)
3. **公共事務參與權指標**(約 30-50 項)
4. **兒童司法人權指標**(約 40-60 項)

合計約 239 項。

## 二、Schema 規格(SQLite + Markdown 雙軌)

每一項指標應記錄:

| 欄位 | 類型 | 說明 |
|---|---|---|
| `indicator_id` | TEXT PK | 如 `NHRC-239-001` |
| `pillar` | TEXT | SDGs / 受教育權 / 公共參與 / 兒童司法 |
| `crc_article` | TEXT[] | 對應 CRC 條文 |
| `co_paragraph` | TEXT[] | 對應 CRC 結論性意見段落號 |
| `nap_action` | TEXT | 對應國家人權行動計畫行動編號 |
| `name_zh` | TEXT | 中文指標名稱 |
| `name_en` | TEXT | 英文指標名稱 |
| `unit` | TEXT | 衡量單位 |
| `data_source_agency` | TEXT | 主管機關(衛福部 / 教育部 / 司法院 etc) |
| `data_url` | TEXT | 指標數值原始來源 URL |
| `latest_value` | TEXT | 最新數值 |
| `latest_date` | TEXT | 最新更新日期 |
| `trend` | TEXT | 趨勢:improving / stable / worsening |
| `nhrc_concern` | TEXT | NHRC 對該指標的監督意見 |
| `related_pi` | TEXT[] | 對應本平台議題卡片 (PI-01 ~ PI-14) |

## 三、人工匯入流程(用戶端動作)

當 NHRC 公布完整指標集 PDF/Excel 後:

```bash
# 1. 下載原始檔到 data/sources/
curl -O https://nhrc.cy.gov.tw/[indicator-set-url]

# 2. 解析為結構化資料(腳本待後續開發)
python3 scripts/parse_nhrc_indicators.py data/sources/NHRC_239_indicators_2026-02.pdf

# 3. 寫入 SQLite + 重建議題卡片連結
python3 scripts/md_to_sqlite.py --rebuild

# 4. 重建 _public/ 議題卡片頁(會自動拉指標)
for f in data/policy_issues/PI-*.md; do
  pi=$(basename "$f" | cut -d'_' -f1)
  python3 scripts/build_single_doc.py --out _public/issues/${pi}.html \
    --strip-frontmatter --schema-type faqpage "$f"
done
```

## 四、議題卡片之指標 backlink 規範

每張 PI 卡片之 frontmatter 增加 `nhrc_indicators` 欄位:

```yaml
nhrc_indicators:
  - id: NHRC-239-042
    name: 校園霸凌通報率(每千人)
    latest_value: "12.3"
    latest_date: 2024-12
    trend: stable
  - id: NHRC-239-043
    name: 校園霸凌處置成立率
    latest_value: "31.87%"
    latest_date: 2025-12
    trend: worsening
    nhrc_concern: NHRC 4,641 件分析顯示 65% 未通報
```

build_single_doc.py 自動偵測此欄位並渲染為「NHRC 對應指標」表格。

## 六、為何此處只有 placeholder

NHRC 監測指標集為 PDF / Excel / 動態網頁形式發布。本平台之 `monitor_updates.py` 已對 NHRC 站點建立指紋追蹤,當該頁面有實質變動時會自動提示。但完整 239 項數值之結構化匯入需要:
1. 用戶下載官方 PDF
2. 人工解析或寫專屬 parser(`scripts/parse_nhrc_indicators.py` 待開發)
3. 確認指標 ↔ PI 之 mapping 是否符合 NHRC 原意

此 placeholder 確保 schema 已就緒,只待數值匯入。
